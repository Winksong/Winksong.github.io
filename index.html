memset

void *memset(void s, int c, size_t n);
s ：指向要填充内存块的起始地址，是void指针类型 ，这使得它能接受任意类型的指针，如数组名、结构体指针等。
c ：要填充的值，虽为int类型，但实际使用时函数会取其低 8 位（即0 - 255范围 ）来填充每个字节。比如想把内存设为全 0，就传0 ；设为全 1，传1 。
n ：需填充的字节数，size_t是无符号整型，常借助sizeof操作符获取内存块大小。

& （按位与）

使用：只有当两个二进制位都为 1 时，结果位才为 1，否则为 0
1.判断奇偶性

3.提取特定位

2.判断 2 的幂次方

^ (异或)

当两个二进制位不同时，结果位为 1；相同时，结果位为 0

删除操作

讨论度的情况

度为0: 直接删
度为1：直接删，再把度为1的子节点替换
度为2：（为了保持删除后保持中序遍历为有序的）转移矛盾，把度为2的节点转化为前驱或后继节点，这些节点必定度为0或1，此时再删去前驱或后继
找前驱和后继的方法：左子树的极大值，右子树的极小值

并查集(不存元素存索引) //代码还没实操

核心操作：合并，查询是否关联
find() union()
实现思路：
Quick Find（顺序结构）
查找效率：O（1）
合并效率：O（N）
核心:是将所有元素进行 ID 分组管理，每个元素对应一个 ID 号
find 的时候，只需要返回这个元素对应的分组 ID
合并的时候 Union (a,b) 时候，将属于 a 分组的元素，都改为 b 分组的 ID 号

Quick Union（树状结构）
查找效率：O（logN）
合并效率：O（logN）
核心:将集合分为根结点和父节点的思想，所有节点保存他的父节点信息，当发现某个节点的父节点就是他自己的时候，这个节点就是根结点
find 操作就是找到这个元素的根结点，判断两个元素的根结点是不是一致，来判断是否连通。
合并操作的时候，Union (a,b) 的时候，不是合并 a 和 b，而是将 a 的根结点和 b 的根结点进行合并

代码优化效率：
基于size算法的优化：再申请一个空间，如果当前索引是根节点，就改存储为这颗树的节点总数，后续要union则小数往大数靠
基于rank算法的优化
